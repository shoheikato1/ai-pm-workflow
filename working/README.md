# working/

In-progress drafts, briefs, and working docs. Anything that doesn't have a permanent home yet.

Files here are named `YYYY-MM-DD-[slug].md`. When a working doc is finalized or no longer active, move it to `working/_archive/`.

The `_staging/` folder at the vault root serves a similar purpose but is session-scoped — Claude auto-saves substantial outputs there before you confirm them. `working/` is for docs you're actively developing across multiple sessions.
