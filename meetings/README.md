# meetings/

Meeting notes, one file per meeting. Named `YYYY-MM-DD-slug.md`.

Generated by the `meeting-ingest` skill. See `templates/meeting-note.md` for the structure.

---

## _context/

Standing context files loaded before recurring meetings. One file per meeting series.

Each context file captures: meeting purpose, recurring participants, standing agenda items, history of past decisions, and things to flag or avoid. Claude loads it automatically before generating a note for that meeting.

Example files to create:

* `_context/weekly-sync-manager.md` — your recurring 1:1 with your manager
* `_context/sprint-ceremonies.md` — standup, planning, retro context
* `_context/[stakeholder]-sync.md` — recurring cross-team syncs

---

## Note format

Every meeting note includes: Summary, Discussion Log, Decisions, Action Items, Open Questions.

Technical meetings also include a **PM Translation Layer**: what the discussion meant in PM terms, and injection points (scope calls engineers made that warrant PM input at the next meeting).
